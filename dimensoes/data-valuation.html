Data Valuation

Data Valuation refere-se ao processo de identificação, mensuração, interpretação e gestão do valor dos dados ao longo de seu ciclo de vida. Essa dimensão parte do pressuposto de que nem todos os dados possuem o mesmo valor analítico, operacional, estratégico, científico ou institucional.

1. Definição

Data Valuation refere-se ao processo de identificação, mensuração, interpretação e gestão do valor dos dados ao longo de seu ciclo de vida. Essa dimensão parte do pressuposto de que nem todos os dados possuem o mesmo valor analítico, operacional, estratégico, científico ou institucional. Em pipelines de dados, especialmente em contextos educacionais, o valor de um ativo de dados não depende apenas de sua existência ou volume, mas de sua capacidade de gerar conhecimento confiável, apoiar decisões, reduzir incertezas, melhorar modelos analíticos, orientar intervenções e produzir impacto mensurável para os atores envolvidos.

No contexto do UnespDataLens-RM, a dimensão Data Valuation tem como objetivo estabelecer critérios para avaliar a relevância e a utilidade dos dados utilizados nos processos de extração, integração, transformação, modelagem, explicabilidade, governança e tomada de decisão. Essa avaliação permite distinguir dados críticos de dados redundantes, dados de alto valor de dados de baixo impacto, dados estratégicos de dados meramente operacionais e dados necessários de dados excessivos ou pouco justificáveis.

Dessa forma, Data Valuation complementa dimensões como qualidade de dados, observabilidade, minimização, governança, proveniência e reprodutibilidade. Enquanto a qualidade de dados busca verificar se os dados são corretos, completos, consistentes e atualizados, a valoração de dados busca responder a uma questão adicional: qual é a contribuição efetiva desse dado para os objetivos analíticos, científicos, educacionais ou institucionais do pipeline?

2. Objetivo da dimensão

O objetivo da dimensão Data Valuation é fornecer mecanismos conceituais, metodológicos e operacionais para estimar o valor dos dados em função de sua utilidade, criticidade, impacto, custo, risco, reutilização e contribuição para a geração de conhecimento.

Essa dimensão busca apoiar decisões como:

  • quais dados devem ser priorizados na coleta;
  • quais fontes possuem maior valor analítico;
  • quais atributos contribuem efetivamente para modelos preditivos ou explicativos;
  • quais dados são essenciais para decisões educacionais;
  • quais dados apresentam baixo valor e alto custo de manutenção;
  • quais dados devem ser preservados, versionados ou descartados;
  • quais ativos de dados justificam maior investimento em governança, qualidade e documentação.

Assim, Data Valuation contribui para tornar o pipeline mais racional, sustentável, auditável e orientado a valor, evitando tanto a subutilização de dados relevantes quanto a manutenção indiscriminada de dados sem finalidade clara.

3. Justificativa

Em projetos de Ciência de Dados, Mineração de Dados Educacionais e Learning Analytics, é comum que grandes volumes de dados sejam coletados, armazenados e processados sem uma avaliação explícita de seu valor. Essa prática pode gerar aumento de complexidade, custos computacionais, riscos de privacidade, redundância, baixa governança e dificuldade de interpretação dos resultados.

A ausência de uma dimensão de Data Valuation pode levar a problemas como:

  • coleta excessiva de dados sem justificativa analítica;
  • manutenção de atributos irrelevantes para o objetivo do estudo;
  • dependência de fontes com baixo impacto decisório;
  • aumento desnecessário de custos de armazenamento e processamento;
  • maior exposição a riscos éticos e legais;
  • dificuldade de explicar por que determinados dados foram utilizados;
  • baixa rastreabilidade entre dados, modelos, métricas e decisões.

No domínio educacional, essa questão é ainda mais sensível, pois os dados podem envolver informações acadêmicas, socioeconômicas, comportamentais e institucionais dos estudantes. Portanto, a valoração dos dados deve considerar não apenas sua contribuição estatística ou preditiva, mas também sua pertinência pedagógica, sua legitimidade institucional, seu potencial de apoio à decisão e seus riscos associados.

Nesse sentido, Data Valuation atua como uma dimensão complementar essencial para evitar que o pipeline seja orientado apenas pela disponibilidade de dados. Em vez disso, propõe que os dados sejam avaliados por sua contribuição efetiva para a compreensão de fenômenos educacionais, para a melhoria de modelos analíticos e para o suporte a decisões responsáveis.

4. Escopo da dimensão

A dimensão Data Valuation abrange a avaliação do valor dos dados em diferentes níveis do pipeline:

4.1 Valor da fonte de dados

Avalia a relevância de cada fonte utilizada no pipeline, considerando sua cobertura, confiabilidade, atualidade, granularidade, custo de obtenção, facilidade de integração e contribuição para os objetivos analíticos.

Exemplos de fontes:

  • sistemas acadêmicos;
  • ambientes virtuais de aprendizagem;
  • questionários socioeconômicos;
  • registros de matrícula;
  • dados de desempenho;
  • dados de frequência;
  • dados administrativos;
  • bases públicas;
  • dados longitudinais.

4.2 Valor do atributo ou variável

Avalia a contribuição de cada variável para análises, modelos, explicações ou decisões. Uma variável pode ter alto valor por ser fortemente associada ao fenômeno estudado, por melhorar o desempenho de um modelo, por favorecer a interpretação dos resultados ou por representar um fator educacionalmente relevante.

Exemplos:

  • idade;
  • renda familiar;
  • desempenho em disciplinas iniciais;
  • frequência;
  • número de reprovações;
  • tempo de acesso ao AVA;
  • curso;
  • modalidade;
  • situação de matrícula.

4.3 Valor do conjunto de dados

Avalia o valor do dataset como ativo analítico, considerando completude, representatividade, potencial de reutilização, documentação, histórico, granularidade, qualidade e aplicabilidade a diferentes problemas.

4.4 Valor do dado para o modelo

Analisa a contribuição dos dados para o desempenho, estabilidade, generalização, explicabilidade e robustez dos modelos analíticos.

4.5 Valor do dado para a decisão

Avalia se o dado contribui para apoiar decisões educacionais concretas, como identificação de estudantes em risco, planejamento de intervenções, revisão curricular, políticas de permanência, ações pedagógicas ou estratégias institucionais.

4.6 Valor do dado em relação ao risco

Considera se o valor analítico de um dado justifica seu custo, sensibilidade, risco de privacidade, risco de viés, risco de uso indevido ou esforço de governança.

5. Componentes da dimensão Data Valuation

A dimensão pode ser organizada nos seguintes componentes:

5.1 Criticidade

Indica o grau de importância do dado para o funcionamento do pipeline ou para a tomada de decisão. Dados críticos são aqueles cuja ausência, erro ou baixa qualidade compromete significativamente os resultados analíticos.

5.2 Utilidade analítica

Representa a contribuição do dado para análises descritivas, diagnósticas, preditivas, prescritivas ou explicativas.

5.3 Impacto decisório

Avalia se o dado contribui para decisões institucionais ou pedagógicas relevantes.

5.4 Potencial explicativo

Indica se o dado ajuda a compreender os fatores associados a determinado fenômeno, como evasão, desempenho, retenção, engajamento ou permanência.

5.5 Contribuição preditiva

Mensura a contribuição do dado para a melhoria do desempenho de modelos analíticos.

5.6 Reutilização

Avalia se o dado pode ser reaproveitado em diferentes estudos, modelos, dashboards, relatórios ou processos institucionais.

5.7 Custo de obtenção e manutenção

Considera o esforço técnico, operacional, computacional e institucional necessário para coletar, tratar, armazenar, atualizar e governar o dado.

5.8 Risco associado

Avalia riscos de privacidade, sensibilidade, discriminação, viés, exposição indevida, uso inadequado ou não conformidade legal.

5.9 Temporalidade do valor

Considera que o valor dos dados pode mudar ao longo do tempo. Dados altamente relevantes em determinado período podem perder valor analítico em outro contexto, especialmente quando ocorrem mudanças curriculares, institucionais, sociais ou tecnológicas.

6. Relação com outras dimensões do modelo

A dimensão Data Valuation possui forte relação com outras dimensões complementares do UnespDataLens-RM.

6.1 Relação com Data Quality

Dados de alto valor precisam ter qualidade suficiente para serem utilizados. Entretanto, um dado pode ter alta qualidade técnica e baixo valor analítico. Por exemplo, um atributo pode estar completo, correto e padronizado, mas não contribuir para os objetivos da análise.

6.2 Relação com Data Minimization

A valoração dos dados apoia a minimização, pois permite identificar quais dados são realmente necessários. Dados de baixo valor e alto risco devem ser questionados quanto à sua permanência no pipeline.

6.3 Relação com Data Governance

A governança deve priorizar ativos de dados com maior valor estratégico, maior criticidade ou maior risco. Data Valuation ajuda a definir níveis diferenciados de controle, documentação, acesso e monitoramento.

6.4 Relação com Proveniência e Linhagem

A valoração depende da capacidade de rastrear a origem, transformação e uso dos dados. Sem proveniência, torna-se difícil justificar o valor atribuído a uma fonte, variável ou dataset.

6.5 Relação com Feature Engineering e Feature Store

A avaliação do valor das variáveis contribui para selecionar, documentar e reutilizar atributos relevantes em modelos analíticos.

6.6 Relação com Data Observability

A observabilidade permite monitorar se ativos de dados de alto valor continuam disponíveis, atualizados e confiáveis ao longo do tempo.

6.7 Relação com Fairness e Bias

Dados de alto valor preditivo podem apresentar riscos de viés ou discriminação. Portanto, a valoração não deve considerar apenas desempenho, mas também implicações éticas e sociais.

8. Métricas associadas

A dimensão Data Valuation pode ser operacionalizada por meio de métricas quantitativas, qualitativas ou híbridas.

8.1 Índice de Valor Analítico do Dado — IVAD

Representa o grau de contribuição de um dado para o objetivo analítico do pipeline.

[ IVAD(d) = \frac{R_a(d) + U_d(d) + P_e(d) + C_p(d)}{4} ]

Em que:

  • (R_a(d)) = relevância analítica do dado;
  • (U_d(d)) = utilidade decisória;
  • (P_e(d)) = potencial explicativo;
  • (C_p(d)) = contribuição preditiva.

Cada componente pode ser avaliado em escala de 0 a 1 ou de 1 a 5.

8.2 Índice de Custo-Risco do Dado — ICRD

Representa o custo e o risco associados ao uso de um dado.

[ ICRD(d) = \frac{C_o(d) + R_p(d) + R_v(d) + E_g(d)}{4} ]

Em que:

  • (C_o(d)) = custo operacional;
  • (R_p(d)) = risco de privacidade;
  • (R_v(d)) = risco de viés;
  • (E_g(d)) = esforço de governança.

8.3 Índice de Valor Líquido do Dado — IVLD

Representa a relação entre valor analítico e custo-risco.

[ IVLD(d) = IVAD(d) - ICRD(d) ]

Quanto maior o IVLD, maior a justificativa para uso e manutenção do dado no pipeline.

8.4 Taxa de Reutilização do Dado — TRD

Mede a frequência com que um dado é reutilizado em diferentes análises, modelos ou produtos analíticos.

[ TRD(d) = \frac{N_{usos}(d)}{N_{processos}} ]

Em que:

  • (N_{usos}(d)) = número de processos em que o dado é utilizado;
  • (N_{processos}) = número total de processos analíticos avaliados.

8.5 Índice de Criticidade do Dado — ICD

Mede o impacto da ausência ou degradação do dado no pipeline.

[ ICD(d) = \frac{I_m(d) + I_d(d) + I_o(d)}{3} ]

Em que:

  • (I_m(d)) = impacto no modelo;
  • (I_d(d)) = impacto na decisão;
  • (I_o(d)) = impacto operacional.

9. Classificação dos dados por valor

A partir dos critérios e métricas, os dados podem ser classificados em quatro categorias.

9.1 Dados de alto valor estratégico

São dados com alta relevância analítica, alto impacto decisório, forte potencial explicativo ou elevada contribuição para modelos. Devem ser priorizados em processos de qualidade, governança, documentação, versionamento e monitoramento.

Exemplo: desempenho acadêmico inicial em estudos de evasão, quando comprovadamente associado ao abandono.

9.2 Dados de valor operacional

São dados necessários para funcionamento do pipeline, integração de bases, identificação de registros ou controle institucional, ainda que não tenham alta contribuição analítica direta.

Exemplo: identificadores internos, códigos de curso, códigos de turma e datas de matrícula.

9.3 Dados de valor contextual

São dados que ajudam a interpretar resultados, segmentar análises ou contextualizar fenômenos, mas que isoladamente podem ter baixo poder preditivo.

Exemplo: modalidade de ensino, turno, polo, ano de ingresso ou tipo de escola de origem.

9.4 Dados de baixo valor ou valor questionável

São dados com pouca contribuição analítica, baixo uso, alto custo, alto risco ou ausência de justificativa clara. Devem ser revisados, descartados ou mantidos apenas mediante justificativa explícita.

Exemplo: atributos coletados por disponibilidade, mas sem relação com os objetivos do estudo ou com decisões institucionais.

10. Processo de aplicação da dimensão

A aplicação da dimensão Data Valuation pode seguir o seguinte processo:

Etapa 1 — Inventariar os ativos de dados

Identificar fontes, datasets, tabelas, atributos, metadados, variáveis derivadas e produtos analíticos utilizados no pipeline.

Etapa 2 — Associar dados aos objetivos do pipeline

Relacionar cada dado aos objetivos analíticos, modelos, dashboards, decisões ou indicadores que ele apoia.

Etapa 3 — Avaliar valor analítico e decisório

Mensurar ou qualificar a contribuição do dado para análise, explicação, predição, decisão e reutilização.

Etapa 4 — Avaliar custo, risco e governabilidade

Identificar esforço de coleta e manutenção, riscos de privacidade, riscos de viés, sensibilidade e exigências de governança.

Etapa 5 — Classificar os dados por valor

Categorizar os dados em alto valor estratégico, valor operacional, valor contextual ou baixo valor.

Etapa 6 — Definir ações de gestão

Decidir se o dado deve ser priorizado, documentado, monitorado, enriquecido, mantido, descartado, anonimizado, restringido ou revisado.

Etapa 7 — Monitorar a evolução do valor

Reavaliar periodicamente o valor dos dados, considerando mudanças no contexto, nos modelos, nos objetivos institucionais e nas políticas educacionais.

12. Exemplo aplicado ao contexto educacional

Em um estudo sobre evasão estudantil, diferentes dados podem apresentar valores distintos.

O atributo “desempenho no primeiro semestre” pode ter alto valor analítico e preditivo, pois pode estar fortemente associado à evasão e contribuir para identificar estudantes em risco. Além disso, possui valor decisório, pois pode orientar ações de acompanhamento acadêmico.

O atributo “renda familiar” pode ter valor contextual e explicativo, pois ajuda a compreender desigualdades e vulnerabilidades, mas também exige cuidado ético e governança adequada, por envolver informação sensível.

O atributo “cor favorita do estudante”, caso eventualmente coletado, tende a ter baixo valor para o problema de evasão, pois dificilmente contribui para análise, predição, explicação ou decisão. Nesse caso, sua manutenção no pipeline seria questionável.

Já o atributo “identificador do estudante” possui valor operacional, pois permite integração e rastreamento dos registros, mas deve ser protegido, anonimizado ou pseudonimizado em etapas analíticas.

Esse exemplo mostra que a valoração dos dados não se limita à contribuição estatística. Ela envolve a relação entre utilidade, custo, risco, explicabilidade, governança e finalidade educacional.

13. Riscos e desafios

A implementação da dimensão Data Valuation apresenta alguns desafios:

  • dificuldade de mensurar valor de forma objetiva;
  • dependência do contexto institucional;
  • variação do valor dos dados ao longo do tempo;
  • risco de supervalorizar dados com alto poder preditivo, mas baixo significado educacional;
  • risco de subvalorizar dados contextuais importantes;
  • conflito entre valor analítico e risco ético;
  • necessidade de participação de especialistas de domínio;
  • dificuldade de comparar dados de naturezas diferentes;
  • ausência de metadados suficientes para avaliar valor;
  • risco de decisões automatizadas baseadas em dados mal interpretados.

Por isso, Data Valuation deve combinar critérios quantitativos e qualitativos, envolvendo tanto métricas computacionais quanto julgamento especializado.

14. Boas práticas

A adoção da dimensão Data Valuation deve considerar as seguintes boas práticas:

  1. atribuir valor aos dados sempre em relação a um objetivo explícito;
  2. evitar considerar volume de dados como sinônimo de valor;
  3. documentar por que cada dado é utilizado;
  4. revisar periodicamente dados de baixo uso ou baixo impacto;
  5. avaliar conjuntamente valor, custo e risco;
  6. priorizar governança para dados críticos e sensíveis;
  7. envolver especialistas educacionais na interpretação do valor;
  8. distinguir valor preditivo de valor pedagógico;
  9. considerar a possibilidade de descarte ou minimização de dados sem finalidade clara;
  10. manter registro histórico das decisões de valoração.

15. Contribuição para o UnespDataLens-RM

A dimensão Data Valuation contribui para o UnespDataLens-RM ao introduzir uma perspectiva orientada a valor na gestão dos dados educacionais. Ela permite que o pipeline não seja avaliado apenas pela quantidade de dados processados ou pelo desempenho dos modelos, mas também pela relevância, utilidade, criticidade, explicabilidade, sustentabilidade e responsabilidade no uso dos dados.

Ao incorporar Data Valuation, a arquitetura passa a oferecer suporte mais robusto para decisões como seleção de fontes, priorização de variáveis, descarte de dados redundantes, investimento em qualidade, definição de políticas de governança e justificativa do uso de dados sensíveis.

Essa dimensão reforça o caráter técnico-metodológico do UnespDataLens-RM, pois transforma a gestão dos dados em um processo avaliável, documentado, rastreável e orientado à geração de valor educacional. Com isso, contribui para que os ativos de dados sejam tratados não apenas como insumos computacionais, mas como recursos estratégicos para a produção de conhecimento, a melhoria dos processos educacionais e o apoio à tomada de decisão baseada em evidências.

16. Síntese da dimensão

A dimensão Data Valuation estabelece critérios para determinar o valor dos dados em um pipeline analítico educacional. Seu foco está em avaliar quanto cada fonte, atributo, dataset ou produto de dados contribui para análise, predição, explicação, decisão, reutilização e impacto institucional, considerando simultaneamente custos, riscos e exigências de governança.

Em síntese, essa dimensão responde às seguintes perguntas:

  • Este dado é necessário?
  • Este dado contribui para o objetivo analítico?
  • Este dado melhora a explicação ou a predição?
  • Este dado apoia alguma decisão educacional?
  • O valor desse dado justifica seu custo e seu risco?
  • Este dado deve ser priorizado, mantido, monitorado, restringido ou descartado?

Com isso, Data Valuation fortalece a capacidade do UnespDataLens-RM de produzir pipelines mais eficientes, responsáveis, sustentáveis e orientados à geração de valor.

Essa dimensão ficou alinhada à lógica de artefatos metodológicos descrita no guia de DSR, especialmente na exigência de explicitar componentes, relações, procedimentos, critérios, entradas, saídas e mecanismos internos do artefato.
